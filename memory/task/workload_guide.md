# 工时评估指南

## 评估参考

| 变更类型 | 参考工时 | 说明 |
|---------|---------|------|
| 条件判断逻辑调整 | 0.5天 | 增加判断分支、修改现有条件，改动量在3-5行以内 |
| 条件判断逻辑调整+防重处理 | 0.5-1天 | 增加分支 + 处理定时任务重循环问题 |
| 条件判断调整+DTO字段限制补偿 | 1天 | 增加分支 + 额外接口调用补偿 DTO 字段缺失 + 单元测试 |
| 新增接口调用 | 0.5-1天 | 调用已有接口，增加状态查询 |
| 完整功能新增 | 2-3天 | 包含接口定义、逻辑实现、测试 |
| 跨模块协调变更 | 2-4天 | 涉及多个模块的联动修改 |

## 历史任务经验

### TASK-001: 授信合同上传规则调整
- 模块：cloudbank-ags
- 变更类型：条件判断逻辑调整 + 定时任务防重处理
- 初步评估：1处分支替换（~3-5行），防重采用"接受重复查询"方案
- 子代理复核发现：主路径（核心查询成功分支）也存在需要修改的位置，实际需改2处分支；主路径 DTO（CreditReqQueryOutput）无 applState 字段，需额外调用 applQueryTargetFacade 获取 applState（方案A）
- 最终评估工时：1天（含单元测试，2处分支修改 + 主路径额外接口调用 + 防重日志埋点）
- 经验：单分支条件调整改动量小，但需注意双路径覆盖风险和 DTO 字段限制。复核阶段往往会发现遗漏的修改点，需根据实际情况调整工时
