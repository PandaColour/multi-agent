# 协议状态机说明

## 协议状态枚举

| 状态码 | 含义 | 说明 |
|-------|------|------|
| CN | 未初始化（需要回填） | AuthFillJob / LoanBackFillJob 拾取的初始状态 |
| CA | 无法创建（作废） | 终态，batchUpdateStatusfailed 会将 CN 置为 CA |
| CI | 等待签署 | 回填成功后（签署模板） |
| WF | 等待填充 | 回填成功后（填充签署模板） |
| DS | 下载成功 | 签署+下载完成，可上传 |
| US | 上传成功 | 上传完成，终态 |
| CS | 签署成功 | 签署完成 |
| RS | - | 终态之一 |
| FR | - | 终态之一 |

## 授信合同（type=AP）状态流转

```
CN（未初始化）
├── 授信通过 + 建档成功 → 回填 → CI/WF → DS → US（上传成功）
├── 掓信通过 + 建档失败 → 保持 CN（不上传，不作废）
└── 授信拒绝 → 保持 CN（新规则：不上传，不作废）
    └── [旧规则：CN → CA（作废）]
```

## 放款合同状态流转

```
CN（未初始化）
├── 放款成功 → 回填 → CI/WF → DS → US（上传成功）
└── 放款失败 → CA（作废）
```

## 关键注意事项

- AuthFillJob 的 SQL 查询条件为 `where status = 'CN'`，所有 CN 状态的协议都会被拾取
- 如果跳过处理但不改变状态，协议仍为 CN，会被下次调度重复拾取，需通过 remark 标记等方式防重
- CA 状态由 batchUpdateStatusfailed SQL 设置，同时写入 remark "放款失败,协议失效"
