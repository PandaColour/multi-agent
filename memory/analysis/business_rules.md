# 业务规则定义

## 授信业务规则

- 授信状态值 | 说明 | 业务含义 |
|-------|------|---------|
| 正常 | 授信通过 | 客户授信审批通过，可正常使用授信额度 |
| 拒绝 | 授信拒绝 | 客户授信审批被拒绝，不可使用授信额度 |
| 审批中 | 审批进行中 | 授信申请正在审批流程中，尚未出结果 |

- 与合同关联规则
| 授信状态 | 合同处理方式 |
|---------|-------------|
| 正常 | 合同可上传行方（需再判断建档状态） |
| 拒绝 | 合同不上传，**不作废** |
| 审批中 | 合同等待，保持当前状态 |

- **授信拒绝场景特殊说明**
    - 授信拒绝后，**授信合同已签署**（穿插签署阶段完成）
    - **不调用额度建档接口**（建档期望100%成功）
    - 定时任务扫描时，查询授信状态为拒绝 → **不上传，不作废**

## 额度建档状态

| 状态值 | 说明 | 业务含义 |
|-------|------|---------|
| 建档成功 | 额度建档完成 | 客户额度已成功建档，可正常使用 |
| 建档失败 | 额度建档失败 | 客户额度建档失败，不可使用额度（终态，不重试） |
| 建档处理中 | 建档流程进行中 | 额度建档流程正在进行，尚未出结果 |
| 无额度记录 | 未查询到额度 | 系统中无该客户的额度信息 |

- 与合同关联规则
| 额度建档状态 | 合同处理方式 |
|-------------|-------------|
| 建档成功 | 合同可上传行方 |
| 建档失败 | 不上传，**不作废** |
| 建档处理中 | 不上传，**不作废** |
| 无额度记录 | 不上传，**不作废** |

## 合同业务规则

| 合同类型 | 签署时机 | 上传规则 | 说明 |
|---------|---------|---------|------|
| 人脸协议 | 授信流程中（穿插） | 签署成功即上传 | 不受授信状态和建档状态影响 |
| 百行授权书 | 授信流程中（穿插） | 签署成功即上传 | 不受授信状态和建档状态影响 |
| 征信授权书 | 授信流程中（穿插） | 签署成功即上传 | 不受授信状态和建档状态影响 |
| **授信合同** | 授信流程中（穿插） | **授信通过+建档成功才上传** | 需查询授信状态和建档状态 |

### 合同状态流转

```
待签署 → 签署成功 → 上传成功
              ↓
           保持不变（授信拒绝/建档失败场景）
```

| 状态 | 说明 | 后续操作 |
|-----|------|---------|
| 待签署 | 合同创建，等待签署 | 签署 → 签署成功 |
| 签署成功 | 合同已签署完成 | 上传 或 保持不变 |
| 上传成功 | 合同已上传行方 | 流程结束 |

### 合同签署规则

- **签署触发方**：获客系统(target-lps)调用核心服务-协议模块
- **签署时机**：获客系统在授信流程中穿插调用各类合同签署
- **签署流程**：获客系统串联，穿插签署各类合同（含授信合同）后，调用额度建档
- **注意**：无论授信通过还是拒绝，授信合同都会在穿插签署阶段签署

### 授信合同上传规则（特殊）

授信合同签署成功后，需满足以下条件才可上传行方：

| 授信状态 | 额度建档状态 | 处理动作 |
|---------|-------------|---------|
| 授信拒绝 | - | **不上传，不作废** |
| 授信通过 | 建档成功 | 上传合同到行方 |
| 授信通过 | 建档失败 | 不上传，**不作废** |
| 授信通过 | 建档处理中 | 不上传，**不作废** |
| 授信通过 | 无额度记录 | 不上传，**不作废** |

**注意**：
- 定时任务扫描签署成功的授信合同后，需先查询授信状态
- 授信拒绝 → 直接不上传，不作废
- 授信通过 → 再查询建档状态决定是否上传

### 其他合同上传规则

人脸协议、百行授权书、征信授权书签署成功后：
- **直接上传**，无需判断授信状态和额度建档状态

## 模块关系说明

### 核心授信模块与协议模块
- **核心授信模块**与**核心协议模块**之间**只有查询接口**
- **没有其他动作**
- 两模块通过**获客系统**串联调用
- 模块之间没有直接的业务依赖

### 核心协议模块与额度模块
- **核心协议模块**与**核心额度模块**之间**只有查询接口**
- **没有其他动作**
- 模块间**没有直接依赖**

### 获客系统与核心服务
- 获客系统(target-lps)作为**业务串联系统**，负责调用核心服务的各模块
- 获客系统在授信流程中**穿插调用**各类合同签署（人脸协议、百行授权书、征信授权书、授信合同）
- 最后调用核心-额度模块进行额度建档
- 额度建档成功后，获客系统的全部流程结束
- 之后由**定时任务驱动**，核心协议模块查询授信状态和建档状态，决定是否上传
- 额度建档在业务逻辑上应**100%成功**，失败仅因网络或异常校验

### 核心服务模块划分

| 模块 | 职责 | 主要功能 |
|-----|------|---------|
| 授信模块 | 授信业务管理 | 授信申请、授信审批、授信状态查询 |
| 协议模块 | 协议管理 | 协议预览、签署、保存、上传 |
| 额度模块 | 额度建档管理 | 额度建档、额度状态查询 |
