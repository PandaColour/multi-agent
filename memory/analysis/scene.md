# 重要场景流程

## 流程阶段说明

**重要**：合同/协议相关的业务分为两个独立阶段，分析需求时需明确涉及哪个阶段：

| 阶段 | 名称 | 触发方 | 涉及动作 |
|-----|------|-------|---------|
| 阶段1 | 授信流程 | 获客系统(target-lps) | **签署** |
| 阶段2 | 定时任务上传 | 定时任务(scheduler) | **上传** |

---

## 场景1：授信流程-穿插签署各类协议

### 触发条件
客户发起授信申请

### 流程时序

```
获客系统                核心服务-协议模块              核心服务-额度模块
   |                         |                              |
   |--穿插签署各类协议------>|                              |
   |  (含授信合同)           |                              |
   |                         |                              |
   |--签署人脸协议--------->|                              |
   |<--签署成功------------|                              |
   |                         |                              |
   |--签署百行授权书------->|                              |
   |<--签署成功------------|                              |
   |                         |                              |
   |--签署征信授权书------->|                              |
   |<--签署成功------------|                              |
   |                         |                              |
   |--签署授信合同--------->|                              |
   |<--签署成功------------|                              |
   |                         |                              |
   |------------------------调用建档----------------------->|
   |                         |                              |--执行建档
   |<-----------------------建档结果-----------------------|
   |                         |                              |
   |      流程结束            |                              |
   |                         |                              |
```

### 业务规则
- 获客系统是**串联流程**的系统，在授信流程中穿插调用各类协议签署
- **穿插签署的协议**：人脸协议、百行授权书、征信授权书、授信合同
- **签署流程不变**：无论授信通过还是拒绝，授信合同都会在穿插阶段签署
- 建档成功后，获客系统的全部流程结束
- **核心授信模块**与**核心协议模块**之间**只有查询接口**
- **核心协议模块**与**核心额度模块**之间**只有查询接口**
- 额度建档在业务逻辑上应**100%成功**，失败仅因网络或异常校验

## 场景2：定时任务-协议上传（按类型区分）

### 触发条件
定时任务(scheduler)定时触发

### 流程时序

```
定时任务        核心服务-协议模块              核心服务-额度模块              行方
   |               |                          |                      |
   |--触发-------->|                          |                      |
   |               |                          |                      |
   |               |--扫描签署成功的协议--------|                      |
   |               |                          |                      |
   |               |--判断协议类型            |                      |
   |               |                          |                      |
   |               |                          |                      |
   |    [其他协议]|--直接上传--------------------------------------->|
   |               |<------------------------上传成功-----------------|
   |               |                          |                      |
   |               |                          |                      |
   |  [授信合同]   |------查询授信状态------->|（授信模块/获客系统） |
   |               |<-------返回授信状态------|                      |
   |               |                          |                      |
   |               |--判断授信状态            |                      |
   |               |                          |                      |
   |  [授信未通过] |--不上传，不作废         |                      |
   |               |                          |                      |
   |               |                          |                      |
   |  [授信通过]   |------查询建档状态------->|                      |
   |               |<-------返回建档状态------|                      |
   |               |                          |                      |
   |   [建档成功]  |-----------上传授信合同--------------------------->|
   |               |<------------------------上传成功-----------------|
   |               |                          |                      |
   | [建档不成功] |--不上传，不作废         |                      |
   |               |                          |                      |
```

### 处理逻辑（授信合同上传规则）

**第一步：查询授信状态**
| 授信状态 | 处理动作 |
|---------|---------|
| 授信未通过 | **不上传，不作废** |
| 授信通过 | 继续查询建档状态 |

**第二步：查询建档状态（仅授信通过时）**
| 额度建档状态 | 处理动作 |
|-------------|---------|
| 建档成功 | 上传协议到行方 |
| 建档失败 | 不上传，**不作废** |
| 建档处理中 | 不上传，**不作废** |
| 无额度记录 | 不上传，**不作废** |

### 其他协议处理

| 协议类型 | 处理动作 |
|---------|---------|
| 人脸协议 | 直接上传 |
| 百行授权书 | 直接上传 |
| 征信授权书 | 直接上传 |

### 业务规则
- 定时任务扫描**签署成功**的协议
- **其他协议**（人脸协议/百行授权书/征信授权书）签署成功后直接上传
- **授信合同**需先查询授信状态，再查询建档状态
- 协议模块通过**查询接口**获取授信状态和额度建档状态
- 模块之间**只有查询接口，无其他动作**

## 场景3：授信拒绝流程

### 触发条件
授信状态 = 拒绝

### 流程时序

```
获客系统          核心服务-协议模块         定时任务
   |                   |                      |
   |--穿插签署各类协议-->|                      |
   |  (含授信合同)      |                      |
   |<--签署成功---------|                      |
   |                   |                      |
   |--授信拒绝         |                      |
   |                   |                      |
   |   流程结束        |                      |
   |   (不调用建档)    |                      |
   |                   |                      |
   |                   |        定时触发------|
   |                   |<----扫描已签署合同---|
   |                   |                      |
   |                   |--查询授信状态        |
   |                   |--授信状态=拒绝       |
   |                   |                      |
   |                   |--不上传              |
   |                   |--不作废              |
   |                   |                      |
```

### 业务规则
- 授信拒绝后，**授信合同已签署**（穿插签署阶段完成）
- **不调用额度建档接口**（建档期望100%成功，只有授信通过才调用）
- 定时任务扫描已签署的授信合同
- 查询授信状态，若为拒绝，则**不上传、不作废**

## 场景4：完整业务流程串联

### 流程概览

```
客户申请
    │
    ▼
授信申请 ──► 授信审批
    │
    ├──[授信拒绝]
    │       │
    │       ▼
    │   穿插签署各类协议（获客系统串联）
    │       │
    │       ├── 签署人脸协议
    │       ├── 签署百行授权书
    │       ├── 签署征信授权书
    │       ├── 签署授信合同
    │       │
    │       ▼
    │   获客系统流程结束（不调用建档）
    │       │
    │       ▼
    │   定时任务扫描授信合同
    │       │
    │       └── 查询授信状态=拒绝 → 不上传，不作废
    │
    └──[授信通过]
            │
            ▼
      穿插签署各类协议（获客系统串联）
            │
            ├── 签署人脸协议
            ├── 签署百行授权书
            ├── 签署征信授权书
            ├── 签署授信合同
            │
            ▼
        额度建档
            │
            ├──[建档成功]──► 获客系统流程结束
            │                    │
            │                    ▼
            │              定时任务扫描授信合同
            │                    │
            │                    └── 授信通过+建档成功 → 上传行方
            │
            └──[建档失败]──► 获客系统流程结束
                                 │
                                 ▼
                           定时任务扫描授信合同
                                 │
                                 └── 授信通过+建档失败 → 不上传，不作废
```

### 需求变更说明

| 变更点 | 原规则 | 新规则 |
|-------|-------|-------|
| 授信拒绝时合同处理 | 作废 | **不上传，不作废** |
| 签署流程 | 不变 | 不变 |
| 建档流程 | 不变 | 不变 |

---

## 模块关系总结

| 模块A | 模块B | 关系 | 说明 |
|------|------|------|------|
| 核心授信模块 | 核心协议模块 | 只有查询接口 | 无其他动作，通过获客系统串联 |
| 核心协议模块 | 核心额度模块 | 相互独立 | 定时任务驱动协议模块查询额度建档状态 |
| 获客系统 | 核心服务 | 串联调用 | 获客系统穿插调用各类协议签署，最后调用额度建档 |
