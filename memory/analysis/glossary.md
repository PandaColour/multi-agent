# 术语表

## 系统术语

| 术语 | 全称 | 说明 |
|-----|------|------|
| cloudbank-LCS | Cloud Bank Core Service | 核心服务，处理核心业务逻辑 |
| target-lps | Target Loan Processing System | 获客系统，负责流程编排 |
| scheduler | Scheduler | 定时任务服务，负责触发定时任务 |
| argus | Argus | 风控系统，负责风险评估 |
| gws | Gateway Service | 网关服务，内部系统与行方的通信网关 |
| 行方 | - | 合作银行，外部系统的统称 |

## 业务术语

### 授信相关

| 术语 | 说明 |
|-----|------|
| 授信 | 银行向客户提供的信用额度 |
| 授信申请 | 客户发起的授信请求 |
| 授信审批 | 对授信申请的审核过程 |
| 授信状态 | 授信的当前状态（正常/拒绝/审批中） |
| 授信额度 | 批准的信用金额 |

### 额度相关

| 术语 | 说明 |
|-----|------|
| 额度建档 | 为客户创建额度档案的过程 |
| 建档节点 | 核心服务中负责额度建档的模块 |
| 额度建档状态 | 额度建档的当前状态（成功/失败/处理中/无记录） |

### 合同相关

| 术语 | 说明 |
|-----|------|
| 合同 | 客户与银行之间的协议文档 |
| 合同签署 | 客户对合同的电子签名确认 |
| 合同上传 | 将签署后的合同上传至行方系统 |
| 合同作废 | 使合同失效，不再具有法律效力 |
| 合同上传任务 | 定时任务，扫描签署成功的合同并判断是否上传 |

#### ⚠️ 易混淆术语辨析

| 术语 | 触发方 | 触发时机 | 说明 |
|-----|-------|---------|------|
| **签署** | 获客系统(target-lps) | 授信流程中穿插调用 | 电子签名动作 |
| **上传** | 定时任务(scheduler) | 签署成功后，定时触发 | 传给行方系统 |

**关键区别**：
- 签署和上传是**两个独立阶段**，不要混淆
- 签署在授信流程中完成（获客系统串联）
- 上传在定时任务中完成（核心服务处理）
- 当需求提到"建档节点"相关变更时，需确认是**签署时机**还是**上传条件**的变更

#### 合同类型

| 术语 | 说明 |
|-----|------|
| 人脸协议 | 人脸识别相关协议，签署成功即上传 |
| 百行授权书 | 百行征信查询授权书，签署成功即上传 |
| 征信授权书 | 征信查询授权书，签署成功即上传 |
| 授信合同 | 授信主合同，**签署成功+建档成功才上传** |

## 状态术语

### 授信状态

| 状态 | 说明 |
|-----|------|
| 正常 | 授信审批通过，额度可用 |
| 拒绝 | 授信审批被拒绝 |
| 审批中 | 授信申请正在审批中 |

### 合同状态

| 状态 | 说明 |
|-----|------|
| 待签署 | 合同已创建，等待客户签署 |
| 签署成功 | 客户已完成签署 |
| 上传成功 | 合同已上传至行方 |
| 作废 | 合同已失效 |

### 额度建档状态

| 状态 | 说明 |
|-----|------|
| 建档成功 | 客户额度已成功建档 |
| 建档失败 | 客户额度建档失败（终态，不重试） |
| 建档处理中 | 额度建档流程进行中 |
| 无额度记录 | 系统中无该客户的额度信息 |
